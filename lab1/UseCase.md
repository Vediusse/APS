
## UC-001: Отображение доступности ресурсов

| Секция | Описание |
| :------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Акторы | Пользователь |
| Приоритет | Must |
| Предусловия | Система функционирует и доступна. <br/> Ресурсы (рабочие места, зоны, оборудование) существуют и зарегистрированы в системе. <br/> Настроены текущие статусы для всех ресурсов. |
| Основной поток событий | 1. Пользователь: Открывает веб-интерфейс или мобильное приложение системы. <br/> 2. Система: Отображает пользовательский графический интерфейс с текущей доступностью всех индивидуальных рабочих мест, специализированных зон и конкретных единиц оборудования (ФТ-001.01.01). <br/> 3. Система: Визуально различает доступность ресурсов с использованием цветовой индикации (ФТ-001.01.02): <br/>    - зеленый — доступно <br/>    - желтый — забронировано <br/>    - красный — используется <br/>    - серый — недоступно <br/> 4. Система: Для каждого ресурса отображает уникальный идентификатор, тип, текущий статус, ближайшее свободное время, а также время окончания активного бронирования (при наличии) (ФТ-001.01.03). <br/> 5. Система: Обновляет статус доступности ресурсов не реже одного раза в 5 секунд (ФТ-001.01.01). |
| Постусловия | Пользовательский интерфейс отображает актуальную информацию о доступности ресурсов. <br/> Состояние системы не изменено. |
| Альтернативные потоки | A-001.1: Отсутствие доступных ресурсов: <br/>   На шаге 2: Система определяет, что нет доступных для бронирования ресурсов. <br/>   1. Система: Отображает все ресурсы как "Забронировано", "Используется" или "Недоступно". <br/>   2. Система: Для каждого ресурса отображает ближайшее свободное время (ФТ-001.01.03). <br/>   Постусловия: Пользовательский интерфейс отображает актуальную информацию о доступности ресурсов, указывая отсутствие доступных. <hr/> A-001.2: Технические неполадки при отображении: <br/>   На шаге 2: Система не может загрузить или корректно отобразить данные о ресурсах. <br/>   1. Система: Отображает сообщение об ошибке загрузки данных или предлагает повторить попытку. <br/>   2. Система: Регистрирует ошибку в системном журнале (ФТ-009.02). <br/>   Постусловия: Пользователь проинформирован о временной недоступности информации; в журнале зафиксирована ошибка. |
| Ссылки на ФТ | ФТ-001.01: Отображение доступности ресурсов. <br/> НФТ-001.01: Время отклика интерфейса бронирования. <br/> НФТ-001.03: Обновление данных. |

## UC-002: Бронирование ресурсов

| Секция | Описание |
| :------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Акторы | Пользователь, Внешняя Платежная Система |
| Приоритет | Must |
| Предусловия | Пользователь зарегистрирован и авторизован в системе (состояние учетной записи: Активна). <br/> Система функционирует и доступна. <br/> Выбранный ресурс находится в состоянии "Доступно" на требуемый период. <br/> Платежная система доступна и корректно настроена для интеграции. |
| Основной поток событий | 1. Пользователь: Просматривает доступные ресурсы (см. UC-001). <br/> 2. Система: Отображает информацию о ресурсах, включая их доступность. <br/> 3. Пользователь: Выбирает желаемый ресурс. <br/> 4. Пользователь: Указывает дату и время начала/окончания бронирования, соблюдая диапазон (30 минут - 24 часа) и дискретность (15 минут) (ФТ-001.02.01, ФТ-001.02.02). <br/> 5. Система: Проверяет, что выбранный ресурс доступен на весь указанный период, и рассчитывает стоимость бронирования в соответствии с тарифами (ФТ-012). <br/> 6. Система: Отображает информацию о бронировании и его стоимость, запрашивает подтверждение для оплаты. <br/> 7. Пользователь: Подтверждает бронирование и инициирует оплату. <br/> 8. Система: Инициирует транзакцию оплаты, взаимодействуя с внешней платежной системой (ФТ-010.02). <br/> 9. Внешняя Платежная Система: Обрабатывает платеж и возвращает системе результат. <br/> 10. Система: Получает подтверждение об успешной оплате от платежной системы (ФТ-010.02). <br/> 11. Система: Фиксирует бронирование в базе данных, устанавливает статус ресурса на "Забронировано", регистрирует финансовую операцию (ФТ-001.02.03, ФТ-010.03). <br/> 12. Система: Отображает подтверждение бронирования в пользовательском интерфейсе. <br/> 13. Система: Отправляет пользователю уведомление о подтверждении бронирования по выбранным каналам (ФТ-005.01.03, ФТ-005.02, ФТ-005.03). <br/> 14. Система: Регистрирует все соответствующие действия в системном журнале (ФТ-009.01). |
| Постусловия | Бронирование создано и имеет статус "Подтверждено". <br/> Выбранный ресурс находится в состоянии "Забронировано". <br/> Финансовая операция об оплате зафиксирована. <br/> Пользователь уведомлен о подтверждении бронирования. |
| Альтернативные потоки | A-002.1: Ресурс стал недоступен до оплаты: <br/>   На шаге 5: Система определяет, что ресурс, выбранный пользователем, стал недоступен для бронирования до момента подтверждения и оплаты. <br/>   1. Система: Уведомляет пользователя о недоступности ресурса. <br/>   2. Система: Предлагает выбрать другой ресурс или скорректировать время бронирования. <br/>   Постусловия: Ресурс остается в текущем состоянии "Доступно". Бронирование не создано. Пользователь возвращается к выбору. <hr/> A-002.2: Отказ в оплате платежной системой: <br/>   На шаге 10: Система получает от внешней платежной системы информацию о неуспешной оплате. <br/>   1. Система: Уведомляет пользователя о неудачной оплате. <br/>   2. Система: Предлагает повторить попытку оплаты или выбрать другой способ оплаты. <br/>   3. Система: Регистрирует неудачную финансовую операцию (ФТ-010.03). <br/>   Постусловия: Бронирование не создано. Ресурс остается в состоянии "Доступно". |
| Ссылки на ФТ | ФТ-001.02: Бронирование ресурсов. <br/> ФТ-005.01.03: Уведомление о подтверждении бронирования. <br/> ФТ-009.01: Регистрация пользовательских действий. <br/> ФТ-010: Интеграция с платёжной системой. <br/> ФТ-012: Управление тарифами и ценами. <br/> НФТ-004.01: Надежность транзакций (ACID). |

## UC-003: Отмена бронирования

| Секция | Описание |
| :------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Акторы | Пользователь |
| Приоритет | Must |
| Предусловия | Пользователь авторизован. <br/> Существует активное бронирование, принадлежащее пользователю, со статусом "Подтверждено". <br/> До запланированного времени начала бронирования осталось не менее 30 минут (ФТ-006.01). <br/> Система функционирует. |
| Основной поток событий | 1. Пользователь: Открывает раздел "Мои бронирования" в пользовательском интерфейсе. <br/> 2. Система: Отображает список активных бронирований пользователя. <br/> 3. Пользователь: Выбирает бронирование для отмены. <br/> 4. Пользователь: Инициирует отмену бронирования. <br/> 5. Система: Проверяет, что до начала бронирования осталось не менее 30 минут (ФТ-006.01). <br/> 6. Система: Изменяет статус бронирования на "Отменено". <br/> 7. Система: Освобождает ресурс, возвращая его в состояние "Доступно". <br/> 8. Система: Рассчитывает сумму возврата средств в соответствии с правилами тарификации (ФТ-006.03) и инициирует операцию возврата через платежную систему (см. UC-030). <br/> 9. Система: Отправляет пользователю уведомление об отмене бронирования (ФТ-005.01.04). <br/> 10. Система: Регистрирует все соответствующие действия в системном журнале (ФТ-009.01). |
| Постусловия | Бронирование имеет статус "Отменено". <br/> Ресурс, связанный с бронированием, находится в состоянии "Доступно". <br/> Осуществлен возврат средств (или запланирован к выполнению) и зафиксирован в финансовых операциях. <br/> Пользователь уведомлен об отмене. |
| Альтернативные потоки | A-003.1: Слишком поздно для отмены: <br/>   На шаге 5: Система определяет, что до начала бронирования осталось менее 30 минут. <br/>   1. Система: Уведомляет пользователя, что отмена невозможна из-за временных ограничений. <br/>   Постусловия: Состояние бронирования и ресурса не изменилось. Пользователь возвращается к списку бронирований. <hr/> A-003.2: Ошибка при возврате средств: <br/>   На шаге 8: Система не может инициировать или завершить возврат средств через платежную систему. <br/>   1. Система: Регистрирует ошибку финансовой операции (ФТ-010.03). <br/>   2. Система: Уведомляет администратора о необходимости ручного вмешательства для возврата средств (ФТ-009.02). <br/>   3. Система: Отправляет пользователю уведомление об отмене бронирования, но указывает на проблему с возвратом средств. <br/>   Постусловия: Бронирование отменено, ресурс свободен, но возврат средств требует ручного разрешения. |
| Ссылки на ФТ | ФТ-005.01.04: Уведомление об отмене бронирования. <br/> ФТ-006.01: Отмена бронирования. <br/> ФТ-006.03: Условия возврата при отмене. <br/> ФТ-009.01: Регистрация пользовательских действий. <br/> ФТ-010.03: Учёт финансовых операций. <br/> НФТ-004.01: Надежность транзакций (ACID). |

## UC-004: Изменение бронирования

| Секция | Описание |
| :------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Акторы | Пользователь |
| Приоритет | Must |
| Предусловия | Пользователь авторизован. <br/> Существует активное бронирование, принадлежащее пользователю, со статусом "Подтверждено". <br/> До запланированного времени начала бронирования осталось не менее 60 минут (ФТ-006.02). <br/> Система функционирует. |
| Основной поток событий | 1. Пользователь: Открывает раздел "Мои бронирования" в пользовательском интерфейсе. <br/> 2. Система: Отображает список активных бронирований пользователя. <br/> 3. Пользователь: Выбирает бронирование для изменения. <br/> 4. Пользователь: Инициирует изменение бронирования. <br/> 5. Система: Проверяет, что до начала бронирования осталось не менее 60 минут (ФТ-006.02). <br/> 6. Пользователь: Изменяет параметры бронирования (время начала, время окончания, тип ресурса, или конкретный ресурс того же типа). <br/> 7. Система: Проверяет доступность выбранного нового ресурса или измененного времени для текущего ресурса (ФТ-006.02). <br/> 8. Система: Рассчитывает новую стоимость бронирования и разницу в оплате (если применимо, учитывая ФТ-012). <br/> 9. Система: Если требуется доплата, предлагает пользователю ее совершить (см. UC-029). Если требуется возврат, инициирует его (см. UC-030). <br/> 10. Система: Обновляет данные бронирования в базе данных, изменяя его статус на "Подтверждено". <br/> 11. Система: Корректирует статус затронутых ресурсов (например, освобождает старый ресурс, если изменен на другой, и резервирует новый). <br/> 12. Система: Отправляет пользователю уведомление об изменении бронирования (ФТ-005.01.05). <br/> 13. Система: Регистрирует все соответствующие действия в системном журнале (ФТ-009.01). |
| Постусловия | Бронирование успешно изменено в соответствии с новыми параметрами и имеет статус "Подтверждено". <br/> Статусы затронутых ресурсов корректно обновлены. <br/> Финансовые операции (доплата или возврат) зафиксированы. <br/> Пользователь уведомлен об изменении. |
| Альтернативные потоки | A-004.1: Слишком поздно для изменения: <br/>   На шаге 5: Система определяет, что до начала бронирования осталось менее 60 минут. <br/>   1. Система: Уведомляет пользователя, что изменение невозможно из-за временных ограничений. <br/>   Постусловия: Состояние бронирования и ресурса не изменилось. Пользователь возвращается к списку бронирований. <hr/> A-004.2: Недоступность нового ресурса/времени: <br/>   На шаге 7: Система обнаруживает, что новый ресурс или выбранное измененное время недоступны. <br/>   1. Система: Уведомляет пользователя о недоступности. <br/>   2. Система: Предлагает выбрать другие параметры бронирования. <br/>   Постусловия: Бронирование не изменено. Ресурс остается в исходном состоянии. Пользователь возвращается к шагу 6. <hr/> A-004.3: Отказ или ошибка при доплате: <br/>   На шаге 9: Пользователь отказывается от доплаты или платежная система отклоняет транзакцию. <br/>   1. Система: Уведомляет пользователя о необходимости доплаты для завершения изменения. <br/>   2. Система: Регистрирует неудачную финансовую операцию (ФТ-010.03). <br/>   Постусловия: Бронирование не изменено. Ресурс остается в исходном состоянии. Пользователь может повторить попытку доплаты или отменить изменение. |
| Ссылки на ФТ | ФТ-005.01.05: Уведомление об изменении бронирования. <br/> ФТ-006.02: Изменение бронирования. <br/> ФТ-009.01: Регистрация пользовательских действий. <br/> ФТ-010: Интеграция с платёжной системой. <br/> ФТ-012: Управление тарифами и ценами. <br/> НФТ-004.01: Надежность транзакций (ACID). |

## UC-005: Получение уведомлений (Системный)

| Секция | Описание |
| :------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Акторы | Пользователь |
| Приоритет | Must |
| Предусловия | У пользователя есть активная учетная запись. <br/> Настройки уведомлений пользователя определены (предпочтительные каналы). <br/> Система должна инициировать уведомление в ответ на определенное событие (например, подтверждение бронирования, его изменение/отмена, напоминание о начале/окончании бронирования, начисление штрафа). <br/> Каналы уведомлений (электронная почта, push-уведомления) доступны. |
| Основной поток событий | 1. Система: Идентифицирует событие, требующее отправки уведомления пользователю (например, успешное бронирование, изменение бронирования, приближение начала бронирования) (ФТ-005.01). <br/> 2. Система: Формирует содержание уведомления, включающее: <br/>    - тип события <br/>    - идентификатор бронирования <br/>    - тип ресурса <br/>    - время начала и окончания бронирования (ФТ-005.03). <br/> 3. Система: Определяет предпочтительные каналы доставки уведомлений для данного пользователя (электронная почта и/или push-уведомления) (ФТ-005.02). <br/> 4. Система: Отправляет уведомление пользователю через выбранные каналы. <br/>    - Для подтверждения/отмены/изменения бронирования: не более 3 секунд после события (ФТ-005.01.03, ФТ-005.01.04, ФТ-005.01.05). <br/>    - Для начала/окончания бронирования: за 15 минут до и по факту окончания (ФТ-005.01.01, ФТ-005.01.02). <br/> 5. Система: Регистрирует факт отправки уведомления в системном журнале (ФТ-009.02). |
| Постусловия | Пользователь получил уведомление о соответствующем событии. <br/> Факт отправки уведомления зафиксирован в журнале. |
| Альтернативные потоки | A-005.1: Недоступность канала уведомления: <br/>   На шаге 4: Система не может доставить уведомление через один из выбранных каналов (например, ошибка отправки email, недоступность мобильного устройства для push-уведомления). <br/>   1. Система: Пытается доставить через альтернативный канал, если он настроен. <br/>   2. Система: Регистрирует ошибку доставки в системном журнале (ФТ-009.02). <br/>   3. Система: Отправляет уведомление администратору о проблеме с доставкой. <br/>   Постусловия: Уведомление доставлено через другой канал или зафиксирована ошибка доставки. <hr/> A-005.2: Отключение уведомлений пользователем: <br/>   На шаге 3: Пользователь отключил все типы уведомлений или конкретный канал в своих настройках. <br/>   1. Система: Проверяет настройки пользователя и определяет, что отправка уведомления по данному каналу не требуется. <br/>   2. Система: Не отправляет уведомление по отключенным каналам. <br/>   Постусловия: Уведомление не отправлено в соответствии с настройками пользователя. |
| Ссылки на ФТ | ФТ-005: Система уведомлений пользователей. <br/> ФТ-009.02: Регистрация системных событий. <br/> НФТ-003.02: Шифрование при передаче (для безопасности уведомлений). |

## UC-006: Авторизация и доступ к ресурсам

| Секция | Описание |
| :------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Акторы | Пользователь |
| Приоритет | Must |

| Основной поток событий | 1. Пользователь: Подходит к точке доступа (вход в помещение коворкинга или забронированное рабочее место). <br/> 2. Пользователь: Сканирует свой персональный QR-код или прикладывает NFC-метку к считывающему устройству (ФТ-003.02). <br/> 3. Система: В реальном времени получает данные считывания и инициирует процесс валидации (ФТ-003.03). <br/> 4. Система: Проверяет наличие действующего бронирования для данного пользователя на текущий момент времени (ФТ-003.03). <br/> 5. Система: Проверяет соответствие забронированного ресурса точке доступа (ФТ-003.03). <br/> 6. Система: При подтверждении всех условий, подает сигнал на разблокировку электронного замка. <br/> 7. Электронный замок: Разблокируется, предоставляя доступ пользователю. <br/> 8. Система: Регистрирует успешную попытку доступа в системном журнале с указанием времени, идентификатора пользователя, типа события и результата (ФТ-003.04, ФТ-009.01). |
| Постусловия | Доступ предоставлен к помещению/ресурсу. <br/> Успешная попытка доступа зафиксирована в журнале. |
| Альтернативные потоки | A-006.1: Отсутствие активного бронирования: <br/>   На шаге 4: Система определяет, что у пользователя нет действующего бронирования на текущий момент времени. <br/>   1. Система: Отказывает в доступе. <br/>   2. Система: Регистрирует неуспешную попытку доступа в системном журнале (ФТ-003.04, ФТ-009.01). <br/>   Постусловия: Доступ не предоставлен. В журнале зафиксирована неудачная попытка. <hr/> A-006.2: Несоответствие ресурса точке доступа: <br/>   На шаге 5: Система определяет, что забронированный ресурс не соответствует текущей точке доступа. <br/>   1. Система: Отказывает в доступе. <br/>   2. Система: Регистрирует неуспешную попытку доступа в системном журнале (ФТ-003.04, ФТ-009.01). <br/>   Постусловия: Доступ не предоставлен. В журнале зафиксирована неудачная попытка. <hr/> A-006.3: Нераспознанный QR-код/NFC-метка: <br/>   На шаге 3: Считывающее устройство не может распознать QR-код или NFC-метку. <br/>   1. Система: Отказывает в доступе. <br/>   2. Система: Регистрирует неуспешную попытку доступа в системном журнале (ФТ-003.04, ФТ-009.01). <br/>   Постусловия: Доступ не предоставлен. В журнале зафиксирована неудачная попытка. |

## UC-007: Оставление отзывов и оценок

| Секция | Описание |
| :------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Акторы | Пользователь |
| Приоритет | Should |
| Предусловия | Пользователь авторизован. <br/> Пользователь завершил бронирование и использовал ресурс. <br/> С момента окончания бронирования прошло не более 24 часов (ФТ-014.01). <br/> Система функционирует. |
| Основной поток событий | 1. Пользователь: Открывает раздел "Мои завершенные бронирования" или страницу конкретного использованного ресурса в пользовательском интерфейсе. <br/> 2. Система: Отображает список завершенных бронирований или информацию о ресурсе. <br/> 3. Пользователь: Выбирает ресурс, для которого хочет оставить отзыв и оценку. <br/> 4. Система: Предоставляет интерфейс для ввода текстового отзыва (до 500 символов) и выбора оценки (по 5-балльной шкале) (ФТ-014.01). <br/> 5. Пользователь: Вводит текст отзыва и выбирает оценку. <br/> 6. Пользователь: Отправляет отзыв. <br/> 7. Система: Сохраняет отзыв и оценку в базе данных, помечая его как ожидающий модерации. <br/> 8. Система: Регистрирует действие пользователя в системном журнале (ФТ-009.01). <br/> 9. Система: Отправляет уведомление администратору о новом отзыве (ФТ-009.02). |
| Постусловия | Отзыв и оценка сохранены в системе и ожидают модерации. <br/> Действие пользователя зафиксировано в журнале. |
| Альтернативные потоки | A-007.1: Истек срок для оставления отзыва: <br/>   На шаге 4: Система определяет, что с момента окончания бронирования прошло более 24 часов. <br/>   1. Система: Уведомляет пользователя, что срок для оставления отзыва истек. <br/>   Постусловия: Отзыв не принят. Пользователь не может оставить отзыв для данного ресурса. <hr/> A-007.2: Слишком длинный отзыв: <br/>   На шаге 6: Пользователь пытается отправить отзыв, превышающий 500 символов. <br/>   1. Система: Выводит сообщение об ошибке валидации длины текста. <br/>   2. Система: Просит пользователя сократить текст отзыва. <br/>   Постусловия: Отзыв не сохранен. Пользователь остается на странице оставления отзыва. |
| Ссылки на ФТ | ФТ-009.01: Регистрация пользовательских действий. <br/> ФТ-014: Система отзывов и оценок. |

## UC-010: Управление ресурсами (CRUD)

| Секция | Описание |
| :------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Акторы | Администратор |
| Приоритет | Must |
| Предусловия | Администратор авторизован в системе (состояние учетной записи: Активна, с административными правами). <br/> Административный интерфейс доступен. |
| Основной поток событий | 1. Администратор: Заходит в административный интерфейс системы. <br/> 2. Администратор: Выбирает раздел "Управление ресурсами". <br/> 3. Система: Отображает список всех существующих физических ресурсов (рабочие места, специализированные зоны, оборудование). <br/> **Подпоток: Создание ресурса** <br/> 4. Администратор: Инициирует создание нового ресурса. <br/> 5. Система: Предоставляет форму для ввода характеристик ресурса: уникальный идентификатор, название, описание, тип, поддерживаемое ПО, физическое местоположение. <br/> 6. Администратор: Заполняет форму и сохраняет новый ресурс. <br/> 7. Система: Создает новый логический ресурс в базе данных и привязывает его к указанным характеристикам (ФТ-007.01, ФТ-007.02, ФТ-007.03). <br/> 8. Система: Регистрирует действие администратора в системном журнале (ФТ-009.02). <br/> **Подпоток: Просмотр/Редактирование ресурса** <br/> 9. Администратор: Выбирает существующий ресурс из списка для просмотра или редактирования. <br/> 10. Система: Отображает текущие характеристики выбранного ресурса (ФТ-007.02). <br/> 11. Администратор: Изменяет необходимые характеристики (например, название, статус, привязку к оборудованию). <br/> 12. Администратор: Сохраняет изменения. <br/> 13. Система: Обновляет данные ресурса в базе данных (ФТ-007.01, ФТ-007.02, ФТ-007.03). <br/> 14. Система: Регистрирует действие администратора в системном журнале (ФТ-009.02). <br/> **Подпоток: Удаление ресурса** <br/> 15. Администратор: Выбирает ресурс для удаления. <br/> 16. Администратор: Подтверждает удаление. <br/> 17. Система: Удаляет логический ресурс из базы данных (ФТ-007.01). <br/> 18. Система: Регистрирует действие администратора в системном журнале (ФТ-009.02). |
| Постусловия | Список ресурсов обновлен (добавлен/изменен/удален). <br/> Изменения в данных ресурсов зафиксированы в системе. <br/> Действия администратора зарегистрированы в журнале. |
| Альтернативные потоки | A-010.1: Некорректные данные при создании/редактировании: <br/>   На шаге 6 (создание) или 11 (редактирование): Администратор вводит некорректные или отсутствующие обязательные данные. <br/>   1. Система: Выводит сообщение об ошибке валидации. <br/>   2. Система: Указывает на некорректные поля для исправления. <br/>   Постусловия: Ресурс не создан/изменен. Администратор остается в форме редактирования/создания. <hr/> A-010.2: Попытка удаления используемого/забронированного ресурса: <br/>   На шаге 17: Администратор пытается удалить ресурс, который в данный момент забронирован или используется. <br/>   1. Система: Выводит предупреждение о невозможности удаления, пока ресурс активен. <br/>   2. Система: Предлагает отменить активные бронирования или удалить ресурс принудительно. <br/>   Постусловия: Ресурс не удален или удален после подтверждения отмены бронирований. Администратор возвращается к списку ресурсов. |
